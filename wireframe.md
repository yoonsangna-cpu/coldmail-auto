# 콜드메일 자동발송 - UI 와이어프레임

---

## 전체 화면 구조

```
+----------------------------------------------------------+
|  사이드바 (좌측 고정)       |   메인 영역 (우측)            |
|                            |                              |
|  - Gmail 로그인 영역        |   Step 1 ~ Step 4            |
|  - 발송 설정               |   (단계별 콘텐츠 표시)         |
|                            |                              |
+----------------------------------------------------------+
```

- 사이드바: Gmail 로그인 + 발송 설정 (항상 표시)
- 메인 영역: 4단계 탭으로 구성, 순서대로 진행

---

## 사이드바

```
+----------------------------+
|  📧 Gmail 로그인            |
|                            |
|  Gmail 주소                |
|  [________________@gmail]  |
|                            |
|  앱 비밀번호                |
|  [********************]    |
|                            |
|  [    연결 테스트    ]      |
|                            |
|  ✅ 연결 성공!              |
|                            |
|----------------------------|
|                            |
|  ⚙️ 발송 설정               |
|                            |
|  발송 간격(초)              |
|  [  3  ] (스팸 방지)        |
|                            |
|  발신자 이름                |
|  [________________]        |
|                            |
+----------------------------+
```

### 설명
- Gmail 주소와 앱 비밀번호를 입력하고 "연결 테스트"를 누르면 인증 확인
- 연결 성공 시 초록색 체크 표시, 실패 시 빨간색 에러 메시지
- 발송 간격은 기본 3초 (Gmail 스팸 차단 방지)
- 발신자 이름은 수신자에게 보이는 이름 (예: "홍길동 / ABC컴퍼니")

---

## 메인 영역 - 4단계 탭

```
+----------------------------------------------------------+
|  [ Step 1: 메일 작성 ]  [ Step 2: 엑셀 업로드 ]           |
|  [ Step 3: 미리보기   ]  [ Step 4: 발송        ]           |
+----------------------------------------------------------+
```

---

## Step 1: 메일 작성

```
+----------------------------------------------------------+
|                                                          |
|  📝 메일 스크립트 작성                                     |
|                                                          |
|  ℹ️ 변수를 넣고 싶은 곳에 {변수명}을 입력하세요.             |
|     예: {회사명}, {담당자}, {직책}                          |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  메일 제목                                                |
|  +----------------------------------------------------+  |
|  | {회사명} 협업 제안드립니다 - ABC컴퍼니                  |  |
|  +----------------------------------------------------+  |
|                                                          |
|  메일 본문                                                |
|  +----------------------------------------------------+  |
|  | 안녕하세요, {담당자}님.                                |  |
|  |                                                      |  |
|  | {회사명}의 {직책}님께 협업을 제안드리고자               |  |
|  | 연락드립니다.                                         |  |
|  |                                                      |  |
|  | 저희 ABC컴퍼니는 ...                                  |  |
|  |                                                      |  |
|  +----------------------------------------------------+  |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  🔖 사용된 변수 목록: {회사명}, {담당자}, {직책}            |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  ⚠️ 빈 데이터 처리                                        |
|                                                          |
|  변수 값이 비어있는 경우 어떻게 처리할까요?                   |
|                                                          |
|  ◉ 기본값으로 대체                                        |
|                                                          |
|  | 변수명   | 기본값        |                              |
|  |----------|--------------|                              |
|  | {회사명}  | [귀사      ] |                              |
|  | {담당자}  | [담당자    ] |                              |
|  | {직책}   | [         ] |                              |
|                                                          |
|  ○ 빈 값이 있는 행은 별도 템플릿 사용                       |
|                                                          |
|  (선택 시 대체 제목/본문 입력란 펼침)                       |
|                                                          |
+----------------------------------------------------------+
```

### 설명
- 제목과 본문에 `{변수명}` 형식으로 자유롭게 변수 삽입
- 입력 즉시 사용된 변수를 자동 감지하여 하단에 표시
- 빈 데이터 처리 옵션:
  - **기본값 대체**: 각 변수별 기본값 설정 (빈 칸이면 해당 변수 부분을 통째로 제거)
  - **별도 템플릿**: 빈 값이 있는 수신자용 제목/본문을 따로 작성

## Step 2: 엑셀 업로드

```
+----------------------------------------------------------+
|                                                          |
|  📂 엑셀 파일 업로드                                      |
|                                                          |
|  +----------------------------------------------------+  |
|  |                                                      |  |
|  |     파일을 여기에 드래그하거나 [파일 선택] 클릭         |  |
|  |     (.xlsx, .xls 지원)                               |  |
|  |                                                      |  |
|  +----------------------------------------------------+  |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  ✅ 파일 업로드 완료: contacts.xlsx (총 52건)              |
|                                                          |
|  📋 데이터 미리보기                                       |
|                                                          |
|  | #  | 이메일            | 회사명     | 담당자 | 직책  |  |
|  |----|-------------------|-----------|--------|------|  |
|  | 1  | kim@company.com   | 삼성전자   | 김철수  | 과장  |  |
|  | 2  | lee@startup.io    | 토스      | 이영희  | 팀장  |  |
|  | 3  | park@firm.co.kr   | 카카오     |  ⚠️   | 대리  |  |
|  | 4  | choi@corp.com     | 네이버     | 최민수  |  ⚠️  |  |
|  | 5  | ...               | ...       | ...    | ...  |  |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  📊 데이터 요약                                           |
|                                                          |
|  - 전체 데이터: 52건                                      |
|  - 정상 데이터 (모든 변수 있음): 48건                      |
|  - ⚠️ 빈 값 포함 데이터: 4건                              |
|  - ❌ 이메일 없음 (발송 불가): 0건                         |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  🔗 변수 연결                                             |
|                                                          |
|  수신 이메일 열 선택: [ 이메일 ▼ ]                         |
|                                                          |
|  | 템플릿 변수  | 연결된 엑셀 열   |                       |
|  |-------------|-----------------|                       |
|  | {회사명}     | [ 회사명 ▼ ]    |  ✅ 자동 매칭          |
|  | {담당자}     | [ 담당자 ▼ ]    |  ✅ 자동 매칭          |
|  | {직책}      | [ 직책   ▼ ]    |  ✅ 자동 매칭          |
|                                                          |
+----------------------------------------------------------+
```

### 설명
- 드래그 앤 드롭 또는 파일 선택으로 엑셀 업로드
- 업로드 즉시 데이터 미리보기 테이블 표시
- 빈 값이 있는 셀은 노란색 경고 아이콘으로 표시
- 변수명과 엑셀 열 이름이 같으면 자동 매칭 (다르면 드롭다운으로 수동 선택)
- 이메일 열은 필수 지정 (발송 대상 주소)

## Step 3: 미리보기

```
+----------------------------------------------------------+
|                                                          |
|  👁️ 발송 미리보기                                         |
|                                                          |
|  전체 52건 중 미리보기 표시                                |
|                                                          |
|  수신자 선택: [ ◀ ]  1 / 52  [ ▶ ]                        |
|                                                          |
|  ┌──────────────────────────────────────────────┐        |
|  │  To: kim@company.com                          │        |
|  │  Subject: 삼성전자 협업 제안드립니다 - ABC컴퍼니  │        |
|  │─────────────────────────────────────────────── │        |
|  │                                                │        |
|  │  안녕하세요, 김철수님.                           │        |
|  │                                                │        |
|  │  삼성전자의 과장님께 협업을 제안드리고자            │        |
|  │  연락드립니다.                                   │        |
|  │                                                │        |
|  │  저희 ABC컴퍼니는 ...                            │        |
|  │                                                │        |
|  └──────────────────────────────────────────────┘        |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  ⚠️ 빈 값이 있는 메일 (4건)                               |
|                                                          |
|  | #  | 수신자             | 빈 변수   | 처리 방식     |   |
|  |----|--------------------|---------| -------------|   |
|  | 3  | park@firm.co.kr    | 담당자   | → "담당자님"   |   |
|  | 4  | choi@corp.com      | 직책    | → (제거됨)     |   |
|  | ...                                               |   |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  ✅ 발송 준비 완료                                        |
|  - 정상 발송: 48건                                        |
|  - 기본값 적용 발송: 4건                                   |
|  - 발송 제외: 0건                                         |
|                                                          |
+----------------------------------------------------------+
```

### 설명
- 좌우 화살표로 각 수신자별 최종 메일 내용을 1건씩 확인
- 실제 변수가 모두 치환된 최종 메일 형태로 표시
- 빈 값이 있는 메일을 별도 테이블로 정리하여 처리 방식 확인
- 하단에 발송 건수 요약

## Step 4: 발송

```
+----------------------------------------------------------+
|                                                          |
|  🚀 메일 발송                                             |
|                                                          |
|  ┌──────────────────────────────────────────────┐        |
|  │  발송 대상: 52건                               │        |
|  │  예상 소요 시간: 약 2분 36초 (간격 3초 기준)      │        |
|  └──────────────────────────────────────────────┘        |
|                                                          |
|  [         ✉️ 발송 시작         ]                         |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  📊 발송 진행률                                           |
|                                                          |
|  [████████████████░░░░░░░░░░░░] 35 / 52  (67%)           |
|                                                          |
|  ✅ 성공: 34건                                            |
|  ❌ 실패: 1건                                             |
|  ⏳ 남은 시간: 약 51초                                    |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  📋 발송 로그 (실시간)                                     |
|                                                          |
|  | 시간     | 수신자             | 상태  | 메모          |  |
|  |----------|--------------------|------|--------------|  |
|  | 14:23:01 | kim@company.com    | ✅   |              |  |
|  | 14:23:04 | lee@startup.io     | ✅   |              |  |
|  | 14:23:07 | park@firm.co.kr    | ✅   | 기본값 적용    |  |
|  | 14:23:10 | error@invalid      | ❌   | 주소 오류     |  |
|  | ...      | ...                | ...  | ...          |  |
|                                                          |
|  ─────────────────────────────────────────                |
|                                                          |
|  (발송 완료 후)                                           |
|                                                          |
|  🎉 발송 완료!                                            |
|                                                          |
|  - 성공: 51건 / 실패: 1건                                 |
|                                                          |
|  [ 📥 결과 CSV 다운로드 ]                                  |
|                                                          |
+----------------------------------------------------------+
```

### 설명
- 발송 시작 버튼 클릭 시 순차적으로 메일 발송
- 실시간 진행률 바와 성공/실패 카운터 표시
- 발송 로그가 실시간으로 테이블에 쌓임
- 발송 완료 후 결과를 CSV로 다운로드 가능 (성공/실패 목록)
- 실패 건은 사유를 함께 표시 (주소 오류, 타임아웃 등)

---

## 사용 흐름 요약

```
Gmail 로그인 (사이드바)
       ↓
Step 1: 메일 제목/본문 작성 + 변수 {회사명} {담당자} 삽입 + 빈값 처리 설정
       ↓
Step 2: 엑셀 업로드 → 데이터 확인 → 변수-열 자동 매칭
       ↓
Step 3: 수신자별 최종 메일 미리보기 확인
       ↓
Step 4: 발송 시작 → 실시간 진행률 → 완료 → 결과 다운로드
```

---

## 디자인 원칙

1. **단순함**: 4단계만 따라가면 완료
2. **안전함**: 미리보기 단계에서 반드시 확인 후 발송
3. **투명함**: 빈 데이터 처리 방식을 명확히 표시
4. **피드백**: 모든 단계에서 성공/실패 상태를 즉시 알려줌
